# diagrams-skills

Este repositorio reúne un conjunto de plantillas y ejemplos para generar diagramas y documentos técnicos a partir de conversaciones o descripciones breves. Está pensado para apoyar la creación rápida de diagramas UML, de procesos, de arquitectura, de bases de datos y otros artefactos visuales útiles en análisis, diseño y documentación de proyectos.

## ¿Qué contiene este proyecto?

- Carpeta `skills/`: archivos Markdown con instrucciones de uso para generar diferentes tipos de diagramas y documentos.
- Carpeta `diagrama-ejemplo/`: ejemplos de archivos `.drawio` listos para abrir en diagrams.net / draw.io.
- Enfoque práctico: convertir requerimientos o contexto textual en salidas visuales editables.

## Tipos de contenido incluidos

El repositorio incluye plantillas para generar:

- Diagramas de casos de uso
- Diagramas de clases
- Diagramas de secuencia
- Diagramas de flujo de programa
- Diagramas de procesos
- Diagramas de arquitectura de sistema
- Diagramas ER
- Diagramas de máquina de estados
- Diagramas Ishikawa
- Líneas de tiempo
- Actas de reunión
- Análisis de brechas
- Historias de usuario
- Informes de avance de proyecto

## ¿Para qué sirve?

Este proyecto es útil cuando se quiere:

- Transformar requisitos o ideas en diagramas claros y profesionales.
- Acelerar la documentación técnica de un sistema o proceso.
- Crear artefactos visuales compatibles con diagrams.net / draw.io.
- Usar instrucciones reutilizables para asistentes de IA o flujos de trabajo de generación de contenido.

## Estructura del repositorio

```text
.diagrams-skills/
├── diagrama-ejemplo/      # ejemplos en formato .drawio
└── skills/                # plantillas y guías de generación
```

## Posible uso

1. Crear un proyecto en ChatGPT y cargar las fuentes de este repositorio, incluyendo los archivos de la carpeta `skills/`.
2. Abrir un chat nuevo y referenciar habilidades como `$acta-reunion-simple`, `$caso-uso-simple`, `$diagrama-arquitectura-sistema`, `$html-interactivo-visual` y otras disponibles.
3. Proporcionar el contexto del proyecto, requisitos o descripción del proceso.
4. Obtener una salida estructurada, textual y visual que pueda usarse directamente en documentación, presentaciones o análisis.
5. Guardar el resultado como archivo `.drawio` o `.html` y abrirlo en herramientas como draw.io o directamente en https://app.diagrams.net/ para visualizar o editarlo.

## Uso recomendado con ChatGPT

La idea de uso más práctica es la siguiente:

- Crear un proyecto de ChatGPT dedicado a diagramas y documentación.
- Cargar como fuentes los archivos de la carpeta `skills/`.
- Usar el chat para invocar las habilidades por nombre, por ejemplo: `$acta-reunion-simple`, `$caso-uso-simple`, `$diagrama-arquitectura-sistema` o `$html-interactivo-visual`.
- Obtener respuestas más consistentes y alineadas con el formato esperado por este repositorio.

## Uso crítico y funcionalidad relevante

Este repositorio no solo sirve para generar contenido visual, sino también para acelerar tareas de análisis y comunicación. Una funcionalidad especialmente útil es la capacidad de convertir información de reunión, requisitos o contexto de negocio en entregables accionables, como:

- actas de reunión claras,
- diagramas de casos de uso listos para compartir,
- arquitectura de sistemas en formato visual,
- y documentos HTML interactivos para presentar información de forma ejecutiva.

## Visualización de diagramas

Para ver o editar los diagramas generados, se recomienda:

- descargar draw.io o diagrams.net en el equipo, o
- utilizar directamente https://app.diagrams.net/ para abrir archivos `.drawio` y trabajar sobre ellos sin instalación adicional.

## Ejemplo de uso

Si se necesita un diagrama de arquitectura, se puede usar la plantilla de `skills/diagrama-arquitectura-sistema.md` para describir el sistema y obtener un diseño base listo para ajustar.

Del mismo modo, para un diagrama de casos de uso, se puede emplear la plantilla de `skills/diagrama-caso-uso.md` con la descripción del módulo o proceso.

## Recomendación

Este repositorio funciona muy bien como base para:

- talleres de análisis y diseño,
- documentación de software,
- generación de artefactos para presentaciones,
- y preparación de materiales para reuniones técnicas.
