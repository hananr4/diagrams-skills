`$# Skill: Generar Documento HTML Interactivo Visual

## Identificador de skill

`$html-interactivo-visual`

## Cuándo usar

Usa `$html-interactivo-visual` cuando quieras transformar información ya creada en esta conversación en un documento HTML profesional, visual e interactivo.

Este skill es útil para:

* Convertir una propuesta, resumen, análisis, acta, plan o documento técnico en una presentación HTML visual.
* Explicar procesos complejos de forma clara.
* Mostrar fases, riesgos, métricas, actividades o decisiones de manera ejecutiva.
* Preparar material para presentar a socios, clientes, inversionistas, gerencia o equipos técnicos.
* Convertir información larga en una experiencia navegable con pestañas, diagramas, tablas y tarjetas visuales.

No usar este skill cuando:

* Solo se necesita un resumen simple en texto.
* Se requiere un documento Word, PDF o PowerPoint.
* La información base todavía no está clara o no ha sido creada.
* Se busca únicamente corregir redacción sin cambiar el formato.

---

## Prompt

Toma la información que acabas de crear y transfórmala en un documento HTML interactivo y muy visual.

Aplica un diseño profesional, agradable, moderno y perfectamente jerarquizado. El resultado debe estar pensado para ser presentado a nivel ejecutivo, comercial o técnico, según el contexto de la información original.

No inventes información. Usa únicamente los datos, conceptos, decisiones, riesgos, fases, métricas y conclusiones que ya estén presentes en el contenido base. Si falta algún dato, déjalo indicado como pendiente o no especificado.

El documento HTML debe incluir obligatoriamente los siguientes elementos:

### 1. Diagramas de flujo en formato SVG

Incluye diagramas de flujo en SVG que muestren claramente:

* El camino principal del proceso.
* Las tareas que se realizan en paralelo.
* Las actividades que se solapan.
* Los puntos de decisión importantes.
* Las dependencias entre fases o áreas.

El SVG debe estar embebido directamente en el HTML, sin depender de archivos externos.

---

### 2. Tarjetas de métricas clave

Organiza los datos o resultados más importantes en tarjetas visuales tipo dashboard.

Las tarjetas deben mostrar, cuando aplique:

* Métricas principales.
* Cantidades relevantes.
* Porcentajes.
* Fechas.
* Duraciones.
* Costos.
* Riesgos críticos.
* Beneficios esperados.
* Indicadores de avance o impacto.

Cada tarjeta debe tener título, valor destacado y una breve explicación.

---

### 3. Sistema de pestañas

Agrupa la información detallada en un sistema de pestañas interactivo para evitar scroll excesivo.

Las pestañas pueden organizarse por:

* Fases.
* Áreas temáticas.
* Roles.
* Procesos.
* Riesgos.
* Decisiones.
* Plan de acción.
* Alternativas.
* Recomendaciones.

Las pestañas deben funcionar con JavaScript simple embebido en el mismo archivo HTML.

---

### 4. Línea de tiempo horizontal

Incluye una línea de tiempo horizontal que represente las fases temporalmente.

Debe mostrar:

* Nombre de cada fase.
* Duración estimada, si existe.
* Orden de ejecución.
* Solapamientos entre actividades, si existen.
* Hitos relevantes.
* Dependencias críticas.

Si no existen fechas o duraciones exactas, usa fases relativas como:

* Inicio.
* Preparación.
* Ejecución.
* Validación.
* Lanzamiento.
* Seguimiento.

No inventes fechas específicas.

---

### 5. Tablas interactivas con código de colores

Genera tablas visuales e interactivas para organizar información importante.

Pueden ser tablas de:

* Riesgos.
* Procesos.
* Actividades.
* Responsables.
* Decisiones.
* Comparativos.
* Requerimientos.
* Plan de acción.
* Costos.
* Priorización.

Aplica código de colores coherente:

* Rojo: impacto alto, riesgo crítico, prioridad alta o bloqueo.
* Amarillo: impacto medio, riesgo moderado o atención requerida.
* Verde: impacto bajo, riesgo controlado, avance positivo o prioridad baja.
* Azul o gris: información neutral, contexto o datos generales.

Incluye filtros o búsqueda simple si la tabla contiene muchos registros.

---

## Requisitos técnicos del HTML

El resultado debe ser un único archivo HTML completo, incluyendo:

* `<!DOCTYPE html>`
* `<html lang="es">`
* `<head>`
* `<style>` embebido
* `<script>` embebido
* Diseño responsive
* Sin dependencias externas obligatorias
* Sin librerías CDN, salvo que se justifique claramente
* Compatible con navegador moderno
* Código limpio, ordenado y comentado cuando sea necesario

---

## Estilo visual esperado

Usa un diseño:

* Profesional.
* Moderno.
* Ejecutivo.
* Limpio.
* Con buena jerarquía visual.
* Con uso moderado de sombras, bordes redondeados y espacios.
* Con colores sobrios y consistentes.
* Fácil de leer en pantalla grande y laptop.

Prioriza claridad sobre decoración.

---

## Estructura sugerida del documento

El HTML debe incluir como mínimo:

1. Encabezado principal con título y subtítulo.
2. Resumen ejecutivo.
3. Tarjetas de métricas clave.
4. Diagrama SVG del flujo principal.
5. Línea de tiempo horizontal.
6. Sistema de pestañas con detalle por tema o fase.
7. Tablas visuales con código de colores.
8. Riesgos, decisiones o puntos críticos, si aplican.
9. Conclusiones o próximos pasos.
10. Pie de página con fecha de generación o nota de uso.

---

## Instrucciones de calidad

Antes de entregar el HTML:

* Verifica que las pestañas funcionen.
* Verifica que las tablas sean legibles.
* Verifica que el SVG se vea correctamente.
* Verifica que el diseño sea responsive.
* Evita texto genérico innecesario.
* No agregues datos inventados.
* No incluyas referencias tipo `[1]`, `[2]`, `[3]` si no existen fuentes reales en el contenido.
* Mantén la información fiel al documento original.
* Si hay información crítica, destácala visualmente.

---
